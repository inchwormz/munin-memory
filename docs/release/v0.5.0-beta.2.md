# Munin v0.5.0-beta.2 Testing Build

Generated: 2026-04-19

## Plain-English Status

Munin is open-source-ready for controlled testing. It is not a finished 1.0
launch and should not be marketed as customer-proven yet.

The binary builds, installs, routes memory-shaped questions, installs skills
for Codex and Claude, reads strategy metrics, runs morning proactivity, and
passes Doctor release checks on this machine.

## What To Prove Before Customer Rollout

1. A fresh checkout can build and install Munin without local-only assumptions.
2. A new Codex session loads the installed Munin skills and follows the resolver.
3. A new Claude session loads the installed Munin skills and follows the resolver.
4. `munin resume`, `munin recall`, `munin nudge`, `munin proactivity`, `munin prove`, `munin doctor`, and `munin hygiene` help in real work, not just tests.
5. Public docs are understandable to someone who has never seen the local repo.

## Release Gate

Required local checks:

```powershell
cargo test --quiet
cargo build --quiet --bin munin
munin install --check-resolvable
munin proactivity status --scope <scope> --format text
munin doctor --scope user --release --repo-root . --format text
```

Optional site-docs parity check:

```powershell
munin doctor --scope user --release --repo-root . --site-root <published-site-root> --format text
```
